# [模块] 职责与使用指南

> 适用版本：JitPack `com.github.acefun29.jswarm:*:1.0.1`  
> Java 包名仍为 `com.jswarm.*`；Maven `groupId` 为 `com.github.acefun29.jswarm`。  
> 能力稳定性以 [`adr/[ADR]module-boundaries-and-public-api.md`](adr/[ADR]module-boundaries-and-public-api.md) 为准。

## 1. 先选依赖（大多数业务只看这里）

| 场景 | 显式引入 | 说明 |
|------|----------|------|
| LangChain4j 编排 | `jswarm-adapter-langchain4j` | **推荐默认**；会传递带上 core / spi / runtime |
| Spring AI（手写装配） | `jswarm-adapter-spring-ai` | 同步 / 流式 `SwarmRunner` |
| Spring Boot 一键装配 | `jswarm-spring-boot-starter` | 自动 `SwarmRunner`；可选带上 observability |
| 只要拓扑类型 | `jswarm-core` | 一般不必单独引；写库/拆模块时用 |
| 自写 Adapter / 兼容测试 | `jswarm-adapter-tck`（+ spi） | 应用业务通常不需要 |
| Micrometer 日志/指标 Advisor | `jswarm-observability-micrometer` | 仅 Spring AI 路径；也可被 starter 可选引入 |

**不会发布到 JitPack（仅源码示例）：** `jswarm-examples`、`jswarm-examples-spring-ai`。

仓库块（所有坐标共用）：

```xml
<repositories>
  <repository>
    <id>jitpack.io</id>
    <url>https://jitpack.io</url>
  </repository>
</repositories>
```

## 2. 分层关系

```
业务代码
    │
    ├─ jswarm-spring-boot-starter          （可选 Boot）
    ├─ jswarm-observability-micrometer     （可选，Spring AI）
    │
    ▼
jswarm-adapter-langchain4j  或  jswarm-adapter-spring-ai
    │
    ▼
jswarm-runtime          唯一模型无关编排状态机
    │
    ▼
jswarm-spi              Canonical message / RunScope / Tool / Error
    │
    ▼
jswarm-core             Swarm / Agent / SwarmContext
```

原则：Adapter 只做 provider 桥接；handoff / delegate / recovery / 生命周期顺序由 **runtime** 统一决定。

## 3. 各模块说明

### 3.1 `jswarm-core`

**职责：** Agent 与 Swarm 拓扑描述；请求级 `SwarmContext`（`{key}` 模板）。零外部依赖。

**何时用：** 所有场景都会间接用到；仅当拆模块/定义拓扑库时才单独依赖。

**关键类型：** `Swarm`、`SwarmBuilder`、`Agent`、`SwarmContext`、`SwarmEvent`

**用法要点：**

```java
Swarm swarm = Swarm.create("customer-service")
        .agent(router).agent(tech)
        .entry("router")
        .handoff("router", "tech")
        .delegate("router", "order")
        .build();

SwarmContext ctx = new SwarmContext();
ctx.put("user_name", "张三"); // 须在该 Agent 首轮 resolve 前写入
```

**注意：**

- `{key}` 只在 Agent 进入主循环 / handoff / delegate 入口 **resolve 一次**；同 Agent 内 tool 里 `ctx.put` **不会**刷新已渲染的 SystemMessage。
- 跨多次 `run()` 的会话历史由**应用自管**（或用 adapter 的 `runWithHistory` + `HistoryStore`）。

---

### 3.2 `jswarm-spi`

**职责：** Provider 无关契约——canonical 消息、RunScope（预算/取消）、工具描述、错误码、HistoryStore、ModelGateway 等。

**何时用：** 自写 Adapter / 高级集成；普通业务经 adapter facade 即可，**通常不必直接依赖**。

**关键包：** `com.jswarm.spi.message`、`spi.run`、`spi.lifecycle`、`spi.error`、`spi.history`

**注意：** 工具必须先变成 `ToolDescriptor` 并进入 registry，再经 scope 授权；旧式“未注册名直接 executor”路径已收紧。

---

### 3.3 `jswarm-runtime`

**职责：** 唯一共享编排状态机（`RunEngine`）：路由、生命周期钩子、recovery、terminal、工具注册与编排工具注入。

**何时用：** 自写 Adapter；业务侧通过 `SwarmRunner` **间接**使用，一般不直接依赖。

**关键类型：** `RunEngine`、`RunInput`、`RunResult`、`RunHandle`、`ToolRegistry`、`OrchestrationTools`

**注意：**

- 框架注入的编排工具名固定为 **`handoff`**、**`delegate`**，用户 `@Tool` **不得同名**。
- Adapter 不得复制模型无关主循环。

---

### 3.4 `jswarm-adapter-langchain4j`

**职责：** LangChain4j 运行时——`JAgent`、`SwarmRunner`、消息编解码、工具桥接、handoff/delegate 过滤。

**Maven：**

```xml
<dependency>
  <groupId>com.github.acefun29.jswarm</groupId>
  <artifactId>jswarm-adapter-langchain4j</artifactId>
  <version>1.0.1</version>
</dependency>
```

**关键类型：** `com.jswarm.adapter.lc4j.JAgent`、`...run.SwarmRunner`、`SwarmRunOptions`、`ExternalToolExecutor`

**用法：**

```java
import com.jswarm.adapter.lc4j.JAgent;
import com.jswarm.adapter.lc4j.run.SwarmRunner;
import com.jswarm.core.Swarm;
import com.jswarm.core.SwarmContext;

JAgent router = JAgent.builder("router", "路由")
        .description("分析意图")
        .instructions("技术问题 handoff 到 tech。当前用户 {user_name}。")
        .model(chatModel)
        .tools(orderTools)   // 可选：LC4j @Tool Bean
        .onEnter(ctx -> { /* ... */ })
        .build();

// 也可：JAgent.fromAiService(...)、fromTools(...)、decorate(...)

Swarm swarm = Swarm.create("cs").agent(router).agent(tech).entry("router")
        .handoff("router", "tech").build();

String reply = SwarmRunner.create(swarm).run("激活码无效", ctx);
// 流式：runner.runStreaming(msg, ctx, listener)
// 多轮：runner.runWithHistory(..., historyStore)
```

**稳定能力：** `builder` / `fromTools` / `fromAiService` / `run` / `runStreaming`  
**注意：** `@SystemMessage` 走 Jswarm `{key}`，不是 LC4j `{{var}}` + `@V`。`ToolProvider` 已弃用，改用 `ExternalToolExecutor`。

---

### 3.5 `jswarm-adapter-spring-ai`

**职责：** Spring AI 运行时——与 LC4j 对称的 `JAgent` / `SwarmRunner` facade。

**Maven：**

```xml
<dependency>
  <groupId>com.github.acefun29.jswarm</groupId>
  <artifactId>jswarm-adapter-spring-ai</artifactId>
  <version>1.0.1</version>
</dependency>
```

**关键类型：** `com.jswarm.adapter.springai.JAgent`、`...run.SwarmRunner`、`ExternalToolExecutor`

**用法：** 与 LC4j 相同模式（注意 import 包名是 `springai`）：

```java
import com.jswarm.adapter.springai.JAgent;
import com.jswarm.adapter.springai.run.SwarmRunner;

JAgent agent = JAgent.builder("tech", "技术支持")
        .description("...")
        .instructions("...")
        .model(chatModel)
        .build();

SwarmRunner.create(swarm).run(userMessage, ctx);
SwarmRunner.create(swarm).runStreaming(userMessage, ctx, event -> { /* Token / 路由事件 */ });
```

**稳定：** `builder` / `fromTools` / `run` / `runStreaming`  
**未实现：** `JAgent.fromAiService(...)` → 抛 `UnsupportedOperationException`（请用 builder / fromTools）。  
**矩阵：** Spring AI **2.0.0** + Spring Boot **4.0.7**（不支持 AI 2.0 + Boot 3.x）。

---

### 3.6 `jswarm-adapter-tck`

**职责：** 两 Adapter 共用的兼容性测试夹具（不是业务运行时）。

**何时用：** 自研 Adapter 时 `extends AdapterCompatibilityTck` 并对齐契约。

**关键类型：** `AdapterCompatibilityTck`、`AdapterHarness`、`TckFixture`

**业务应用：不要引入。**

---

### 3.7 `jswarm-observability-micrometer`

**职责：** Spring AI 路径上的可选可观测性——脱敏日志 Advisor、Micrometer 指标 Advisor。

**Maven：**

```xml
<dependency>
  <groupId>com.github.acefun29.jswarm</groupId>
  <artifactId>jswarm-observability-micrometer</artifactId>
  <version>1.0.1</version>
</dependency>
```

**用法要点：**

- 依赖本模块（或经 starter 可选引入）。
- `jswarm.logging.enabled=true` 时可自动挂 logging Advisor。
- `SwarmMetricsAdvisor` 常需自行提供 `MeterRegistry` 的 `@Bean`。

**注意：** 仅服务 Spring AI / Boot 矩阵；LC4j 路径不走这套 Advisor。

---

### 3.8 `jswarm-spring-boot-starter`

**职责：** Spring Boot 自动配置——在容器中有 **唯一** `Swarm` Bean 时装配 `SwarmRunner` / `SwarmRunOptions`。

**Maven：**

```xml
<dependency>
  <groupId>com.github.acefun29.jswarm</groupId>
  <artifactId>jswarm-spring-boot-starter</artifactId>
  <version>1.0.1</version>
</dependency>
```

**用法要点：**

1. 引入 starter（传递 spring-ai adapter）。
2. 提供 **一个** `Swarm` `@Bean`（多 Swarm 时不会自动装配 runner）。
3. 配置项示例：`jswarm.max-turns`、`max-recovery-attempts`、`max-delegate-depth`、`model-timeout`、`logging.enabled`。

**关键类型：** `JswarmAutoConfiguration`、`JswarmProperties`

## 4. 编排语义速查（两 Adapter 共用）

| 概念 | 行为 |
|------|------|
| `handoff` | 目标 Agent 接管；保留对话历史；替换并 resolve SystemMessage |
| `delegate` | 子循环执行任务；结果作为 tool result 回到原 Agent |
| 生命周期 | `onEnter` / `onExit` / `onDelegateEnter` / `onDelegateExit`；钩子抛异常中断 run |
| Agent 实例 | 同 id 单例，服务所有请求；`SwarmContext` 请求级隔离 |
